### Battle Debugger

**UC-6.0.1**: Run multiple battles between two Pokemon
- **Actor**: Developer
- **Goal**: Test Pokemon matchups over many battles
- **Steps**:
  1. Select player Pokemon (or Random)
  2. Select enemy Pokemon (or Random)
  3. Set level (1-100)
  4. Set number of battles (1-10000)
  5. Click "Run"
  6. View statistics (win/loss/draw, move usage, status effects)

**UC-6.0.2**: Analyze move usage patterns
- **Actor**: Developer
- **Goal**: Understand which moves Pokemon use most often
- **Steps**:
  1. Run battles (UC-6.0.1)
  2. View "Move Usage" tab
  3. See move usage counts and percentages per Pokemon

**UC-6.0.3**: Analyze status effect rates
- **Actor**: Developer
- **Goal**: Understand status effect application rates
- **Steps**:
  1. Run battles (UC-6.0.1)
  2. View "Status Effects" tab
  3. See status effect counts and percentages per Pokemon

### Move Debugger

**UC-6.0.4**: Test move damage ranges
- **Actor**: Developer
- **Goal**: Verify move power ranges
- **Steps**:
  1. Select move
  2. Select attacker Pokemon
  3. Select target Pokemon
  4. Set level
  5. Set number of tests (1-10000)
  6. Click "Run"
  7. View damage statistics (average, min, max, median)

**UC-6.0.5**: Test status effect chances
- **Actor**: Developer
- **Goal**: Verify status effect application rates
- **Steps**:
  1. Configure move test (UC-6.0.4)
  2. View "Status Effects" tab
  3. See status effect application rates

**UC-6.0.6**: Analyze move action generation
- **Actor**: Developer
- **Goal**: Understand what actions moves create
- **Steps**:
  1. Configure move test (UC-6.0.4)
  2. View "Actions" tab
  3. See all action types generated by the move

### Type Matchup Debugger

**UC-6.0.7**: Calculate type effectiveness
- **Actor**: Developer
- **Goal**: Verify type chart correctness
- **Steps**:
  1. Select attacking type
  2. Select defender primary type
  3. Select defender secondary type (or None)
  4. View effectiveness result

**UC-6.0.8**: View complete type chart
- **Actor**: Developer
- **Goal**: See effectiveness against all types
- **Steps**:
  1. Select attacking type
  2. View "Complete Type Chart" table
  3. See effectiveness against all 18 Pokemon types

## New Debuggers (6.1-6.4)

### 6.1: Stat Calculator Debugger

**UC-6.1.1**: Calculate Pokemon stats with default configuration
- **Actor**: Developer
- **Goal**: See default stat values for a Pokemon
- **Steps**:
  1. Select Pokemon species
  2. Set level
  3. Click "Calculate"
  4. View all calculated stats (HP, Attack, Defense, SpAttack, SpDefense, Speed)

**UC-6.1.2**: Calculate stats with custom Nature
- **Actor**: Developer
- **Goal**: See how Nature affects stats
- **Steps**:
  1. Select Pokemon species
  2. Set level
  3. Select Nature (e.g., Adamant)
  4. Click "Calculate"
  5. View stats with Nature multiplier applied

**UC-6.1.3**: Calculate stats with custom IVs
- **Actor**: Developer
- **Goal**: See how IVs affect stats
- **Steps**:
  1. Select Pokemon species
  2. Set level
  3. Set IVs (0-31 per stat)
  4. Click "Calculate"
  5. View stats with custom IVs

**UC-6.1.4**: Calculate stats with custom EVs
- **Actor**: Developer
- **Goal**: See how EVs affect stats
- **Steps**:
  1. Select Pokemon species
  2. Set level
  3. Set EVs (0-252 per stat, max 510 total)
  4. Click "Calculate"
  5. View stats with custom EVs

**UC-6.1.5**: Compare two different configurations
- **Actor**: Developer
- **Goal**: Compare stats between different builds
- **Steps**:
  1. Configure first Pokemon (species, level, Nature, IVs, EVs)
  2. Configure second Pokemon
  3. Click "Compare"
  4. View side-by-side comparison

**UC-6.1.6**: View stat calculation breakdown
- **Actor**: Developer
- **Goal**: Understand how each component contributes to final stat
- **Steps**:
  1. Configure Pokemon (UC-6.1.1)
  2. View "Stats Table" tab
  3. See base stat, IV, EV, nature multiplier, final value for each stat

### 6.2: Damage Calculator Debugger

**UC-6.2.1**: Calculate damage for a move
- **Actor**: Developer
- **Goal**: See final damage for a move
- **Steps**:
  1. Select attacker Pokemon
  2. Select defender Pokemon
  3. Select move
  4. Set level
  5. Click "Calculate"
  6. View final damage and HP percentage

**UC-6.2.2**: View damage pipeline step-by-step
- **Actor**: Developer
- **Goal**: Understand how damage is calculated
- **Steps**:
  1. Configure damage calculation (UC-6.2.1)
  2. View "Pipeline Steps" tab
  3. See each step: Base Damage, Critical Hit, Random Factor, STAB, etc.
  4. See multiplier and running total for each step

**UC-6.2.3**: View damage range
- **Actor**: Developer
- **Goal**: See min/max damage with random factor
- **Steps**:
  1. Configure damage calculation (UC-6.2.1)
  2. View "Damage Range" tab
  3. See minimum damage (0.85x random factor)
  4. See maximum damage (1.0x random factor)

**UC-6.2.4**: Force critical hit
- **Actor**: Developer
- **Goal**: Test damage with guaranteed critical hit
- **Steps**:
  1. Configure damage calculation (UC-6.2.1)
  2. Check "Force Critical Hit"
  3. Click "Calculate"
  4. View damage with critical hit multiplier

**UC-6.2.5**: Use fixed random value
- **Actor**: Developer
- **Goal**: Test damage with specific random factor
- **Steps**:
  1. Configure damage calculation (UC-6.2.1)
  2. Set fixed random value (0-1)
  3. Click "Calculate"
  4. View damage with specific random factor

**UC-6.2.6**: Test with different weather
- **Actor**: Developer
- **Goal**: See how weather affects damage
- **Steps**:
  1. Configure damage calculation (UC-6.2.1)
  2. Select weather condition
  3. Click "Calculate"
  4. View damage with weather modifier

### 6.3: Status Effect Debugger

**UC-6.3.1**: Apply persistent status
- **Actor**: Developer
- **Goal**: Test persistent status effects
- **Steps**:
  1. Select Pokemon
  2. Select persistent status (Burn, Paralysis, Poison, Sleep, Freeze)
  3. Click "Apply"
  4. View status effects and stat modifications

**UC-6.3.2**: Apply volatile status
- **Actor**: Developer
- **Goal**: Test volatile status effects
- **Steps**:
  1. Select Pokemon
  2. Check volatile status (Confusion, Flinch, etc.)
  3. Click "Apply"
  4. View volatile status effects

**UC-6.3.3**: View stat modifications from status
- **Actor**: Developer
- **Goal**: Understand how status affects stats
- **Steps**:
  1. Apply status (UC-6.3.1)
  2. View "Stat Modifications" tab
  3. See stat changes (e.g., Burn reduces Attack by 50%, Paralysis reduces Speed by 50%)

**UC-6.3.4**: View damage per turn
- **Actor**: Developer
- **Goal**: See damage from Poison/Burn per turn
- **Steps**:
  1. Apply Poison or Burn (UC-6.3.1)
  2. View "Damage Per Turn" tab
  3. See damage amount per turn

**UC-6.3.5**: Test status interactions
- **Actor**: Developer
- **Goal**: Understand which statuses can/cannot be applied together
- **Steps**:
  1. Apply first status (UC-6.3.1)
  2. Try to apply second status
  3. View "Interactions" tab
  4. See which combinations are valid/invalid

**UC-6.3.6**: Test status removal
- **Actor**: Developer
- **Goal**: Test conditions for status removal
- **Steps**:
  1. Apply status (UC-6.3.1)
  2. Trigger removal condition (switch, heal, etc.)
  3. View status removal result
